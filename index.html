Bienvenido

Se debe realizar un aplicativo web que explique qué es la calidad de software y contextualice su aplicación en proyectos de entornos educativos y en ambientes reales. Aquí verás los beneficios de aplicar buenas prácticas, cómo medir calidad y cómo priorizar mejoras técnicas y pedagógicas.

Beneficios clave

  • Mejora experiencia del estudiante y retención.
  • Reduce fallos en producción.
  • Facilita mantenimiento y escalabilidad.

Normas

ISO/IEC 25010, ISO/IEC 27001, IEEE 829 e ISO/IEC 12207 forman la base para definir dimensiones, controles y procesos. Para proyectos educativos, enfatizamos privacidad, integridad y usabilidad adaptativa para estudiantes.

Modelos

Estándares

Linters, políticas de calidad de código, gestión de dependencias, CI/CD y pruebas automatizadas son esenciales. Recomendamos ESLint, Prettier, OWASP dependency-check y pipelines en GitHub Actions para proyectos de la CUN.

Código

Ejemplo de función JS que calcula la puntuación de funcionalidad (documentada y portable):

function scoreFunc(covReqPct, testsPct){
  const cov = Math.max(0, Math.min(100, covReqPct));
  const tests = Math.max(0, Math.min(100, testsPct));
  return cov * 0.6 + tests * 0.4; }

El código del aplicativo está escrito en JS puro y es fácil de integrar en pipelines y pruebas unitarias.

Conclusiones

El uso del modelo permite cuantificar y comparar la calidad entre versiones y proyectos. La evaluación facilita la toma de decisiones y la priorización de mejoras en entornos educativos, donde la usabilidad y la seguridad de datos son críticas.

Recomendaciones

  1. Aumentar cobertura de pruebas unitarias a >80%.
  2. Implementar monitoreo y alertas para reducir fallos.
  3. Programar pruebas de usabilidad periódicas.
  4. Automatizar revisiones de seguridad de dependencias.
  5. Mejorar documentación y pipelines para despliegue reproducible.
